# hardware_risks

## 上电红线

- 首次上电必须限流，按资料建议从 24V、0.2A 限流起步。
- 接电机前先空载 PWM，示波器确认同桥臂上下管 Gate 不重叠，并有约 500ns 死区。
- 先确认 3.3V、VREG、REG12、VS、EN/nFAULT、DRV 输出，再谈闭环。

## STDRIVE101 易错点

- STDRIVE101 没有给 MCU 供电的 VDD 逻辑输出，不要给不存在的 VDD 引脚接 3.3V。
- REG12/VREG 只能按手册接规定电容到地，不能当外部供电脚。
- SCREF/VDS 阈值必须按最终目标电流、MOS Rds(on)、放大/比较链路重新核算；早期 33k/20k 之类方案存在误导风险，不能不复核就流片。
- DT/MODE、EN/nFAULT、VDS 保护、欠压/过流保护必须按数据手册和最终审查意见锁定。

## 电流采样与版图

- 低侧三电阻采样必须 Kelvin 走线。
- OPAMP/PGA 内部反馈相关引脚不能乱接外部网络。
- VINP/VINM、采样电阻、模拟地、电源地要区分高电流回路和小信号回路。
- 四层板应保留完整 GND 平面，功率回路短而粗，采样线远离开关节点。

## 接口与命名

- NUCLEO 接口必须是真实可插接的 Morpho/Arduino 物理连接，不是只画网络名。
- 24V 输入、U/V/W 输出、保险丝/PTC、TVS、反接保护、VS 去耦必须逐项核对。
- Hall 信号命名不能与 IA/IB/IC 电流采样混用。

## 资料冲突处理

如果 V9、早期采购清单、硬件审查意见之间冲突，以“最新审查意见 + 实测验证 + 数据手册”优先。
Codex 在回答硬件问题时默认先问：电源限流多少、示波器看到了什么、nFAULT 状态如何、SCREF/VDS 是否重新算过。
