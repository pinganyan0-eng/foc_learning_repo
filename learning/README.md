# Learning Loop

This folder is the persistent learning memory for the STM32G474 FOC project.

Use it to keep Codex honest about what the user actually understands, where the user gets stuck, and what should be reviewed next.

## Files

- `LEARNING_STATUS.md`: current learning state and active focus.
- `NEXT_LESSON.md`: short next-lesson execution card with progress checkpoint, P0/P1/P2 review priority, acceptance criteria, and Codex handoff.
- `MASTERY_MAP.md`: evidence-based map of demonstrated strengths, active transfer gaps, and do-not-claim-yet areas.
- `session_notes.md`: append-only lesson and Q&A summaries.
- `weak_points.md`: active weak points, evidence level, and repair plan.
- `review_queue.md`: spaced review queue and next checks.
- `question_bank.md`: teach-back prompts and small exercises.

## Evidence Levels

- L0: Not introduced.
- L1: Recognized after explanation.
- L2: Can restate in own words.
- L3: Can solve a guided example.
- L4: Can solve a similar problem independently.
- L5: Can apply it in firmware, hardware, or debugging work.
- L6: Can teach it back and catch edge cases.

## Rule

After any teaching, explanation, debugging guidance, or homework review, update this folder when there is new evidence about understanding, confusion, mistakes, or next review work.
