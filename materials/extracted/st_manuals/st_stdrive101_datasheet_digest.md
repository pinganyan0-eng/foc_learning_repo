## Device Role

STDRIVE101 is the project power-stage gate-driver IC. It is a triple half-bridge gate driver for external N-channel MOSFETs, intended for three-phase brushless motor drive. It belongs to the power-stage safety boundary, so it must not be treated like a normal MCU peripheral.

## Key Datasheet Facts For This Project

| Topic | Datasheet Fact | P2/P3 Meaning |
| --- | --- | --- |
| Supply range | Recommended `VS` operating range is 5.5 V to 75 V; absolute maximum `VS` is 78 V. | No 24 V or power-stage action is allowed in P2. Before P3, the board review must prove the `VS` and power-stage `VM` relationship. |
| Gate-drive supply | `REG12` is the 12 V linear regulator output and gate-driver supply. Datasheet guidance includes 50 mA max total regulator consumption and external capacitance. | Check `REG12` capacitor, load budget, and whether anything external is drawing from it. |
| Logic interface | Logic inputs support 3.3 V / 5 V control domains; recommended input range is 0 V to 5 V. | MCU logic compatibility is plausible, but each signal still needs pull, reset, and fault-state review. |
| Drive modes | `DT/MODE` selects the control strategy: ENx/INx with adjustable deadtime, or INHx/INLx with interlocking when shorted to ground. | MCSDK PWM output style must match the physical `DT/MODE` strap. Do not configure TIM1 complementary outputs blindly. |
| Deadtime | In ENx/INx mode, deadtime is set by the `DT/MODE` resistor. The application example shows `RDT = 50 kOhm` for about 570 ns. | Treat any deadtime value as board-specific. Do not copy it into firmware without power-board schematic and gate waveform review. |
| nFAULT | `nFAULT` is an open-drain fault output. It is forced low for overcurrent, VDS monitoring protection, thermal shutdown, and REG12 UVLO. | Route to an MCU safe fault path. Current P2 draft uses `PB12/TIM1_BKIN` as candidate, but board pull-up voltage and active-low behavior must be verified. |
| Overcurrent comparator | `CP` is the overcurrent comparator input. | The shunt/signal-conditioning path and comparator threshold must be reviewed before any power test. |
| VDS monitoring | `SCREF` sets the VDS monitoring threshold; the protection assumes the driver supply `VS` and power-stage supply `VM` are at the same voltage. If `VM` is lower than `VS`, the protection may false-trigger unless disabled per datasheet guidance. | This is a mandatory hardware-review item before P3. It affects whether nFAULT behavior can be trusted during bring-up. |
| Thermal protection | Thermal shutdown forces nFAULT low and the datasheet describes auto-restart after hysteresis. | Fault logs must distinguish thermal, UVLO, VDS, and overcurrent causes when hardware stages begin. |
| Typical application | ST shows a three-phase motor application with triple shunt configuration, nFAULT pull-up, bootstrap capacitors, REG12 capacitors, CP filtering, and SCREF divider. | Useful review checklist, not a copy-paste schematic. Compare against the actual power-board EDA/netlist. |

## No-Power Review Checklist Before Using This IC In P3

- Confirm the exact `VS` and `VM` connection and prove the datasheet requirement is not violated.
- Confirm `REG12` capacitor values, placement, and total current budget.
- Confirm `nFAULT` is open-drain, has a safe pull-up voltage, and reaches the STM32 fault input path selected in CubeMX/MCSDK.
- Confirm whether the board uses ENx/INx or INHx/INLx mode, and make TIM1/MCSDK output style match it.
- Confirm the `DT/MODE` resistor or ground strap and decide whether deadtime is generated by the driver, by TIM1, or by both.
- Confirm `CP`, shunt, op-amp/signal-conditioning, and comparator threshold network.
- Confirm `SCREF` network and whether VDS monitoring is enabled or intentionally disabled.
- Confirm bootstrap capacitors and gate-drive routing for `GHSx`, `GLSx`, `OUTx`, and `BOOTx`.
- Confirm `STBY` default state and wake-up sequence.
- Keep all of the above as no-power review until the phase gate explicitly allows current-limited hardware bring-up.

## Current Project Implications

- `PB12/TIM1_BKIN` remains the preferred STM32-side nFAULT candidate for the P2 draft.
- The board-side nFAULT pull-up, polarity, and route still need EDA/netlist proof.
- The driver mode selected by `DT/MODE` may change what the MCSDK TIM1 pin map should mean. This must be checked before generating or trusting an MCSDK project.
- VDS monitoring cannot be assumed safe unless `VS`, `VM`, and `SCREF` are reviewed together.
