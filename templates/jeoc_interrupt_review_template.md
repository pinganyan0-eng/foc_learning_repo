# JEOC 中断代码审查模板

## 结论

- 结果：通过 / 有条件通过 / 不通过
- 审查对象：文件、函数、提交号
- 风险等级：P0 停止上电 / P1 必改 / P2 建议 / P3 记录

## 必查项

| 项目 | 通过 | 证据 |
| --- | --- | --- |
| JEOC 由 TIM1_TRGO2 触发，与 PWM 中心对齐 |  |  |
| ISR 内没有 `printf`、`HAL_Delay`、JSON 解析、WebSocket、malloc/free |  |  |
| ADC injected 数据读取顺序与 IA/IB/IC 对应关系正确 |  |  |
| 电流偏置校准只在安全状态执行，不在运行中乱改 |  |  |
| Clarke/Park、PI、SVPWM 更新在预算周期内完成 |  |  |
| 过流、欠压、nFAULT、采样异常能快速置故障标志 |  |  |
| 故障态会禁止 PWM 或拉低使能，不只打印日志 |  |  |
| 与主循环共享变量使用 `volatile`/原子快照/临界区策略 |  |  |
| 速度目标来自已校验命令，ISR 不直接解析 UART 数据 |  |  |
| CORDIC/FMAC 使用失败时有软件回退或错误处理 |  |  |

## 审查提示词

```text
请按 STM32G474 无感 FOC 项目的 JEOC 中断代码审查模板审查以下代码。
重点找实时性、采样时序、状态机、故障保护、共享变量和不该出现在 ISR 的阻塞操作。
输出按 P0/P1/P2/P3 分级，每条必须给文件/函数/触发条件/修复建议。
```

## 固定进团队工作流

- 每次改 `FOC_CurrController`、ADC injected、TIM1、PWM、故障保护、速度目标入口，都必须跑本模板。
- 审查结果和修复记录保存到 `workflow/reviews/`。
- P0/P1 未关闭前，不允许做 24V 接电机测试。
