# Definition of Done

本文件定义不同任务类型的完成标准。任务完成不是“做了动作”，而是有产出、有证据、能复盘、能恢复。

## 学习任务

### 必须产出

- 学习主题、当前理解、薄弱点和下一步练习。
- 必要时更新 `learning/weak_points.md` 或 `learning/review_queue.md`。
- 与项目阶段相关的入口或材料链接。

### 可选产出

- 学习卡片、问题队列、例题、截图或小测结果。
- `learning/session_notes.md` 中的简短记录。

### 不允许伪完成的情况

- 只说“已学习”，没有说明学到了什么。
- 没有记录薄弱点或下一步检查。
- 把未验证的理解写成项目事实。

### 最小验收证据

- 一条可追溯的学习记录，或一个明确的弱点/复习项。
- 如果没有新增弱点，也要说明“无新增弱点”的依据。

## 工程代码任务

### 必须产出

- 明确的修改范围和文件列表。
- 构建、测试或静态检查结果。
- 对接口、状态机或行为变化的说明。

### 可选产出

- 最小复现实验、串口日志、截图、单元测试或回归测试。
- 回滚说明。

### 不允许伪完成的情况

- 没有测试就宣称可用。
- 修改 CubeMX、MCSDK、ESP-IDF 生成配置却没有说明来源和风险。
- 修改实时控制、PWM、Gate、保护阈值或硬件参数但没有阶段闸门。

### 最小验收证据

- 命令结果、测试结果或明确的未运行原因。
- `git diff` 能解释每个改动的目的。

## 实验记录任务

### 必须产出

- 实验日期、目的、环境、步骤、结果、异常和下一步。
- 证据文件路径，例如日志、截图、波形、照片或仪器读数。
- 结论和假设分开写。

### 可选产出

- 失败原因树、复测计划、对比表、答辩可用素材。

### 不允许伪完成的情况

- 只有结论，没有原始证据。
- 没有说明实验条件。
- 把单次异常当成稳定结论。

### 最小验收证据

- `experiments/YYYY-MM-DD_topic/` 下的一条记录，能让下一次复测复现条件。

## 硬件审查任务

### 必须产出

- 审查范围、来源文件、风险点和待复核项。
- 对 24V、PWM、Gate、nFAULT、电流采样、SCREF/VDS、Hall/SMO、电机接入的风险等级判断。
- 是否需要回到 `workflow/phase_gate_checklist.md` 的结论。

### 可选产出

- Datasheet 链接、原理图页码、BOM 行号、计算草稿、检查清单。

### 不允许伪完成的情况

- 没有 Datasheet 或原理图依据就确认安全。
- 让用户直接上电、接电机或改保护阈值。
- 把用户口述器件清单当作已复核设计。

### 最小验收证据

- 一份风险清单，列出证据来源、未确认项和禁止动作。

## 文档/答辩任务

### 必须产出

- 文档、提纲、讲稿、图表或素材池更新。
- 每个关键结论对应来源、实验、截图、日志或项目事实。
- 未验证卖点标记为假设或待验证。

### 可选产出

- 答辩问答、演示脚本、图示草案、PPT 大纲。

### 不允许伪完成的情况

- 夸大实测效果。
- 把未完成工程写成已完成。
- 使用无来源参数或 AI 生成结论冒充证据。

### 最小验收证据

- 更新后的文档路径和关键事实来源说明。

## 仓库维护任务

### 必须产出

- 新增或更新的工作流、索引、状态页或维护脚本说明。
- `CURRENT_STATUS.md` 或 `docs/file_map.md` 的入口更新。
- 维护命令结果。

### 可选产出

- 检索索引刷新、自动化契约更新、任务包或检查清单模板。

### 不允许伪完成的情况

- 只新增文件但没有索引入口。
- 不运行要求的维护命令，也不说明原因。
- 混入固件控制逻辑、工程配置或硬件参数修改。

### 最小验收证据

- 文件列表、命令结果、`git status` 摘要和建议 commit message。
